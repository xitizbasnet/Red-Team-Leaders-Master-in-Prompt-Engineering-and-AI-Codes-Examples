# Prompt Templates Library — Module 02

## 30+ Ready-to-Use Professional Prompt Templates

---

## How to Use This Library

Each template uses placeholders in `[BRACKETS]` that you replace with your specific information. Templates are organized by category and include the key components: role, context, instructions, specifics, and format parameters.

## Category 1: Content Creation

### Template 1.1 — Blog Post

```
Role: You are a professional content writer specializing in [INDUSTRY/NICHE].

Task: Write a blog post on the topic: "[TOPIC TITLE]"

Context:
- Publication: [BLOG/WEBSITE NAME]
- Target audience: [AUDIENCE DESCRIPTION]
- Goal: [INFORM/PERSUADE/EDUCATE/ENTERTAIN]
- SEO target keyword: [KEYWORD]

Requirements:
- Length: [WORD COUNT] words
- Include an engaging hook in the first paragraph
- Use [NUMBER] subheadings (H2)
- Include [NUMBER] practical examples or case studies
- End with a clear call-to-action: [CTA DESCRIPTION]

Tone: [PROFESSIONAL/CASUAL/TECHNICAL/CONVERSATIONAL]

Do NOT include: [EXCLUSIONS — e.g., competitor mentions, pricing, jargon]
```

## Category 2: Analysis and Research

### Template 2.1 — SWOT Analysis

```
Perform a SWOT analysis for the following:

Subject: [COMPANY/PRODUCT/STRATEGY]
Industry: [INDUSTRY]
Context: [RELEVANT BACKGROUND]
Competitive landscape: [KEY COMPETITORS]

Format:
| Category | Items |
|----------|-------|
| Strengths | 4-5 bullet points |
| Weaknesses | 4-5 bullet points |
| Opportunities | 4-5 bullet points |
| Threats | 4-5 bullet points |

After the table, provide:
1. Top strategic priority (based on the SWOT)
2. Biggest risk to address immediately
3. Recommended next step

Be specific — avoid generic points that could apply to any company.
```

### Template 2.2 — Competitive Analysis

```
Role: You are a competitive intelligence analyst.

Analyze [COMPANY/PRODUCT] against these competitors:
1. [COMPETITOR 1]
2. [COMPETITOR 2]
3. [COMPETITOR 3]

Comparison dimensions:
- [DIMENSION 1 — e.g., Pricing]
- [DIMENSION 2 — e.g., Features]
- [DIMENSION 3 — e.g., Market share]
- [DIMENSION 4 — e.g., Customer satisfaction]
- [DIMENSION 5 — e.g., Technology]

Format: Comparison table followed by a 200-word analysis identifying
the key competitive advantages and vulnerabilities of [COMPANY/PRODUCT].

Conclude with 3 specific strategic recommendations.
```

### Template 2.3 — Data Interpretation

```
Analyze the following data and provide insights:

Data:
"""
[PASTE DATA — tables, numbers, CSV, etc.]
"""

Analysis requirements:
1. Summarize the key trends in 3-5 bullet points
2. Identify [NUMBER] notable anomalies or outliers
3. Calculate [SPECIFIC METRICS — growth rate, average, etc.]
4. Provide [NUMBER] actionable insights
5. Highlight any data quality concerns

Present findings as:
- Summary table of key metrics
- Narrative analysis (200-300 words)
- Recommendations (numbered list)

Audience: [WHO WILL READ THIS]
```

### Template 2.4 — Market Research Brief

```
Role: You are a market research analyst.

Create a market research brief on: [TOPIC/MARKET]

Required sections:
1. Market Size & Growth (current and projected 5-year)
2. Key Players (top 5 with estimated market share)
3. Customer Segments (3-4 distinct segments with profiles)
4. Trends (5 emerging trends shaping this market)
5. Barriers to Entry (for a new entrant)
6. Opportunities (3 underserved niches or gaps)

Context: [WHY THIS RESEARCH IS NEEDED]
Geography: [REGION/COUNTRY]
Time horizon: [SHORT/MEDIUM/LONG TERM]
Length: [WORD COUNT] words

Note: Clearly distinguish between data-backed claims and estimates.
State the date or source for any statistics cited.
```

### Template 2.5 — Root Cause Analysis

```
Perform a root cause analysis on the following problem:

Problem statement: [DESCRIBE THE PROBLEM]
When it started: [DATE/TIMEFRAME]
Impact: [QUANTIFY THE IMPACT]
Affected areas: [WHAT/WHO IS AFFECTED]

Apply the following methods:
1. **5 Whys Analysis**: Starting from the problem, ask "Why?" five
   times to reach the root cause
2. **Fishbone Diagram** (text-based): Categorize potential causes
   under: People, Process, Technology, Environment, Materials, Methods
3. **Contributing Factors Matrix**: Rate each factor by
   Likelihood (H/M/L) and Impact (H/M/L)

Conclude with:
- The most likely root cause
- 3 corrective actions (short-term, medium-term, long-term)
- Preventive measures to avoid recurrence
```

## Category 3: Technical and Development

### Template 3.1 — Code Review

```
Review the following code for quality, security, and performance:

```[LANGUAGE]
[PASTE CODE HERE]
```

Review focus areas:
1. Security vulnerabilities (injection, XSS, auth issues)
2. Correctness (logic errors, edge cases, race conditions)
3. Performance (complexity, memory, N+1 queries)
4. Maintainability (readability, naming, SOLID principles)
5. Testing (test coverage gaps, untested edge cases)

For each finding, provide:
- Severity: CRITICAL / WARNING / SUGGESTION
- Line(s): Reference to the specific code
- Issue: Brief description
- Fix: Concrete code suggestion

End with an overall assessment (1-2 sentences) and a summary count
of issues by severity.
```

### Template 3.2 — API Documentation

```
Generate API documentation for the following endpoint:

Endpoint: [METHOD] [URL PATH]
Description: [WHAT IT DOES]
Authentication: [AUTH METHOD]
Rate limit: [LIMIT]

Request:
- Headers: [LIST REQUIRED HEADERS]
- Path parameters: [LIST WITH TYPES]
- Query parameters: [LIST WITH TYPES AND DEFAULTS]
- Body schema: [JSON SCHEMA OR DESCRIPTION]

Responses:
- 200: [SUCCESS RESPONSE SCHEMA]
- 400: [VALIDATION ERROR SCHEMA]
- 401: [AUTH ERROR SCHEMA]
- 404: [NOT FOUND SCHEMA]
- 500: [SERVER ERROR SCHEMA]

Include:
- A complete cURL example
- A Python (requests) example
- A JavaScript (fetch) example
- Common error scenarios and troubleshooting tips

Format: Markdown with code blocks
```

### Template 3.3 — Technical Design Document

```
Role: You are a senior software architect.

Create a technical design document for: [FEATURE/SYSTEM]

Context:
- Current system: [DESCRIBE EXISTING ARCHITECTURE]
- Team size: [NUMBER] engineers
- Timeline: [DEADLINE]
- Constraints: [TECHNICAL/BUDGET/RESOURCE CONSTRAINTS]

Document structure:
1. Overview (what and why)
2. Goals and Non-Goals
3. Proposed Design (with architecture diagram description)
4. API Design (endpoints, schemas)
5. Data Model (entities, relationships)
6. Security Considerations
7. Scalability Considerations
8. Testing Strategy
9. Migration Plan (if applicable)
10. Open Questions
11. Timeline and Milestones

Length: 1500-2500 words
Audience: Engineering team and technical leadership
```

### Template 3.4 — Bug Report Analysis

```
Analyze the following bug report and provide a diagnosis:

Bug report:
"""
[PASTE BUG REPORT — symptoms, steps to reproduce, environment, etc.]
"""

Provide:
1. Most likely root cause (with reasoning)
2. Top 3 possible causes ranked by likelihood
3. Recommended debugging steps (ordered by efficiency)
4. Suggested fix for the most likely cause
5. Regression test to add after the fix
6. Any related issues that should be checked

If information is missing from the bug report, list what additional
information would help diagnose the issue.
```

### Template 3.5 — Database Query Generation

```
Generate SQL queries for the following requirements:

Database: [PostgreSQL/MySQL/SQLite/etc.]
Schema:
"""
[PASTE TABLE DEFINITIONS OR DESCRIBE SCHEMA]
"""

Required queries:
1. [QUERY 1 DESCRIPTION]
2. [QUERY 2 DESCRIPTION]
3. [QUERY 3 DESCRIPTION]

For each query:
- Write the SQL with comments explaining the logic
- Include an execution plan hint (what indexes would help)
- Note the expected time complexity for large datasets
- Provide a test case (sample data + expected result)

Constraints:
- Optimize for [PERFORMANCE/READABILITY]
- [AVOID/USE] subqueries
- Maximum acceptable execution time: [TIME] on [DATA SIZE]
```

## Category 4: Business and Strategy

### Template 4.1 — Executive Summary

```
Write an executive summary for the following:

Document/Project: [NAME]
Audience: [C-SUITE/BOARD/INVESTORS/TEAM]
Purpose: [INFORM/PERSUADE/REQUEST APPROVAL/UPDATE]

Key information to include:
- Background: [BRIEF CONTEXT]
- Main finding/recommendation: [THE KEY POINT]
- Supporting data: [2-3 DATA POINTS]
- Impact: [WHAT THIS MEANS FOR THE ORGANIZATION]
- Ask: [WHAT YOU NEED FROM THE AUDIENCE]

Format: 3-4 paragraphs, under [NUMBER] words
Tone: [CONFIDENT/CAUTIOUS/URGENT/NEUTRAL]
Start with: The most important conclusion (inverted pyramid style)
End with: Clear next steps or decision needed
```

### Template 4.2 — Meeting Agenda

```
Create a meeting agenda for the following:

Meeting title: [TITLE]
Duration: [TIME]
Attendees: [LIST WITH ROLES]
Meeting type: [STATUS UPDATE/DECISION-MAKING/BRAINSTORM/PLANNING]
Objective: [WHAT MUST BE ACCOMPLISHED]

Format:
| Time | Topic | Owner | Objective |
|------|-------|-------|-----------|

Include:
- Pre-meeting preparation requirements for attendees
- Decision items clearly marked
- Time allocated for each topic
- 5-minute buffer for overruns
- Follow-up action items template at the bottom

Keep total agenda items to [NUMBER] or fewer.
```

### Template 4.3 — OKR Generation

```
Role: You are a strategy consultant helping set quarterly OKRs.

Create OKRs for [TEAM/DEPARTMENT/COMPANY].

Context:
- Company stage: [STARTUP/GROWTH/MATURE]
- Last quarter performance: [SUMMARY]
- Strategic priorities: [LIST 2-3 PRIORITIES]
- Team size: [NUMBER]
- Quarter: [Q1/Q2/Q3/Q4 YEAR]

Generate:
- [NUMBER] Objectives (ambitious but achievable)
- [NUMBER] Key Results per Objective (measurable, time-bound)

Format:
**Objective 1:** [Qualitative, inspirational statement]
- KR 1.1: [Quantitative metric] from [baseline] to [target] by [date]
- KR 1.2: [Quantitative metric] from [baseline] to [target] by [date]
- KR 1.3: [Quantitative metric] from [baseline] to [target] by [date]

Requirements:
- Key Results must be measurable with specific numbers
- Include a mix of leading and lagging indicators
- Stretch goals should be achievable 70% of the time
- Align with the stated strategic priorities
```

## Category 5: Education and Training

### Template 5.1 — Lesson Plan

```
Create a lesson plan for teaching [TOPIC].

Learner profile:
- Level: [BEGINNER/INTERMEDIATE/ADVANCED]
- Background: [RELEVANT PRIOR KNOWLEDGE]
- Age group: [AGE RANGE]
- Class size: [NUMBER]

Lesson structure:
1. Learning objectives (3-5 measurable outcomes)
2. Hook/opener (5 minutes — engage curiosity)
3. Core content (presentation with examples)
4. Guided practice (structured exercise)
5. Independent practice (hands-on activity)
6. Assessment (check understanding)
7. Summary and next steps

Duration: [MINUTES/HOURS]
Materials needed: [LIST]
Assessment method: [QUIZ/PROJECT/DISCUSSION/PRACTICAL]

Include:
- Discussion questions for each major section
- Common misconceptions to address
- Differentiation strategies for varying skill levels
```

### Template 5.2 — Quiz/Assessment Generator

```
Create a [NUMBER]-question assessment on [TOPIC].

Assessment type: [MULTIPLE CHOICE/SHORT ANSWER/ESSAY/MIXED]
Difficulty level: [BEGINNER/INTERMEDIATE/ADVANCED]
Target audience: [AUDIENCE]

Requirements:
- [NUMBER] questions at recall level (remembering facts)
- [NUMBER] questions at comprehension level (understanding concepts)
- [NUMBER] questions at application level (using knowledge)
- [NUMBER] questions at analysis level (comparing, evaluating)

For each question include:
- The question
- [For MC: 4 options with one correct, plausible distractors]
- The correct answer
- A brief explanation of why the answer is correct
- The concept being tested

Format: Numbered list with answers in a separate "Answer Key" section
at the end.
```

### Template 5.3 — Concept Explainer

```
Explain [CONCEPT] for someone who [KNOWLEDGE LEVEL].

Structure:
1. One-sentence definition
2. Analogy from everyday life
3. How it works (3-5 step explanation)
4. Why it matters (practical relevance)
5. Common misconception + correction
6. One-paragraph summary

Constraints:
- No jargon without immediate definition
- Every abstraction must have a concrete example
- Reading level: [GRADE LEVEL]
- Length: [WORD COUNT] words
```

## Category 7: Communication and Persuasion

### Template 7.1 — Pitch Deck Outline

```
Create a pitch deck outline for [PURPOSE: investor pitch / sales pitch / internal proposal].

Company/Product: [NAME]
Problem: [THE PROBLEM YOU SOLVE]
Solution: [YOUR SOLUTION]
Target market: [MARKET DESCRIPTION]
Traction: [KEY METRICS]
Team: [KEY TEAM MEMBERS]
Ask: [WHAT YOU ARE REQUESTING]

Generate a [NUMBER]-slide outline:
For each slide provide:
- Slide title
- 3-4 key bullet points
- Speaker notes (2-3 sentences of what to say)
- Suggested visual element

Follow the [SEQUOIA / GUY KAWASAKI / AIRBNB] pitch structure.
```

### Template 7.2 — Objection Handling Guide

```
Generate an objection handling guide for [PRODUCT/SERVICE/PROPOSAL].

Context: [SELLING SITUATION]
Target audience: [WHO YOU ARE SELLING TO]

For each of the following [NUMBER] common objections:

1. [OBJECTION 1 — e.g., "It's too expensive"]
2. [OBJECTION 2 — e.g., "We already have a solution"]
3. [OBJECTION 3]
4. [OBJECTION 4]
5. [OBJECTION 5]

Provide:
- **Acknowledge**: How to validate the concern (1 sentence)
- **Reframe**: How to shift the perspective (1-2 sentences)
- **Evidence**: A specific proof point, case study, or data
- **Close**: A question or statement to move forward

Keep each response under 60 words (for natural conversation use).
```
